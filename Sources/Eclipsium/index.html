Spline-Sync Mechanics

Objective: Create a seamless mechanic tying together two distinct “body forms” (e.g., miniature planet and full planet) so that player inputs and environmental interactions remain fluid during transitions.

Requirements Gathering & Design

  • Collaborated with level designers to define how each body form should react to world geometry, physics, and input.
  • Sketched flow diagrams showing player state transitions, required data hand‐offs (pose, velocity, animation state), and edge cases (mid-air switches, collision overlaps).

Implementation Highlights

  • State-Driven Progresser Bridge: Built a lightweight C# class that listens for form-switch events and copies over transform, velocity, and animation parameters in real time.
  • Dual-Rig Sync: Extended Unity’s Spline system to mirror root transforms between two rigs, ensuring the secondary rig (celestial planet) perfectly overlaps the primary during blending.
  • Smooth Interpolation: Wrote a coroutine-based smoothing routine that interpolates position, rotation, and scale over a configurable duration. Prevented jitter when switching on complex geometry and improved play-feel.
  • Edge-Case Handling: Added checks to detect if the target form’s collider would intersect world geometry. If so, the switch is deferred until a safe frame to avoid “stuck” states.

Results:

  • Achieved an instantaneous, polished planet-shift feel.
  • Improved player feedback and immersion.

Shader “Consumption” Rework & Effects

Objective: Overhaul the original “darkness” damage‐over‐time effect and enhance it with tentacle visuals for a more menacing feel.

Core Gameplay Tuning

  • Redesigned damage tick timing and radius falloff so players could see and react to the encroaching darkness.
  • Added anticipatory VFX cues—consumption by tendrils—before darkness took hold, giving clear visual telegraphing.

Tentacle Additions

  • Created a runtime system to spawn tentacle meshes around the player’s periphery when darkness activated.
  • Each tentacle spawns at a random rotation around the player; the tendrils (shader) grow exponentially based on the player’s proximity.
  • Blended tentacle shader values in/out based on distance, so they appear to “reach” and then recede if the player escapes the radius.

Custom Spline, Mesh Deforming & Gravity Shifting

Custom Spline - Core Classes & Data Structures

  • SplinePoint: Holds position, tangent handles, and a “roll” angle to control twist.
  • SplineSegment: Connects two points with cubic Bézier interpolation; exposes GetPoint(t), GetTangent(t), and GetNormal(t) methods.
  • Spline: Manages an ordered list of points, builds an array of segments on edit, and maintains a cached lookup table of uniform‐length samples for fast runtime queries.

Editor Integration

  • Leveraged Unity’s OnSceneGUI to draw handles in Scene view—points can be clicked, dragged, or deleted.
  • Custom inspector buttons (“Add Curve”, “Undo Last”) call repository commands to modify the point list and auto-commit changes to version control.

Mesh Deformation Along Splines

  • Built a deformation system that “bends” arbitrary meshes to follow curves.
  • Vertex Remapping Algorithm: Each deformable mesh uses a UV channel where U=distance along spline (0-1) and V=lateral offset.
  • Sample Spline: For each vertex, sample position = spline.GetPoint(u) and tangent = spline.GetTangent(u).
  • Construct Frenet Frame: Using tangent and the spline’s up vector (or “roll”), compute a normal and binormal to form a full local coordinate frame.
  • Apply Offset: Use the vertex’s original lateral offset (V) along binormal so the mesh wraps smoothly around the path.

Gravity Reorientation via Surface Normals

  • For custom gravity moments (walking up walls, inverted slopes), wrote a routine to dynamically shift the gravity vector to align with spline normals.
  • Normal Extraction: At a player-attach point (e.g. feet), sample the nearest spline normal via GetNormal(u). Blend normals if on multiple splines.
  • Gravity Vector Rotation: Set desiredDown = –sampledNormal. Slerp between current gravity and desiredDown over 0.25 s for a gentle world “rotation.”
  • Controller Integration: Feed updated gravity into controller.ApplyGravity(gravityVector) and rotate player’s root so “up” opposes gravity.
  • Edge Cases & Stability: Clamp interpolation speed mid-air to avoid camera flip.

Editor Tool: Level Selector

During ECLIPSIUM’s development, we followed a strict scene structure to avoid merge conflicts and maximize productivity. To simplify opening all required scenes for each level, I created a custom Editor tool.

Architectural Overview

  • Uses a ScriptableObject approach to store required scenes and any additional ones.
  • Inherits from EditorWindow to open a custom window and allow user interaction.
  • Opens and closes scenes based on the user’s selection in that window.
  • Features a fast, intuitive search bar to filter available ScriptableObjects.
